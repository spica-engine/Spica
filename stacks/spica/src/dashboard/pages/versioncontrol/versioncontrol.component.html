Version Control

Quick Start Guide

First commit 

git add .
git commit -m "first commit"
git branch -M main
git remote add origin https://<username>:<access_token_or_password>@<repo_address>
git push -u origin main

Clone repository 

git remote add origin https://<username>:<access_token_or_password>@<repo_address>
git fetch --all
# perform it for each branch that will be switched to
# local changes will be overwritten from now
git checkout -b <branch_name> origin/<branch_name> --force
git reset --hard origin/<branch_name>
# in case there are untracked directories, files
git clean -d -f

Adding specific files 

1. Module scoped      
git add /<module_name>/*
# git add /bucket/*

2. Resource scoped      
git add /<module_name>/<resource_id>/*
# git add /bucket/6267978d69d28205eaa76baa/*

3. Sub-module scoped      
git add /<module_name>/<resource_id>/<submodule_name>*

- Environment variables 
# git add /function/6267978d69d28205eaa76baa/env* 

- Index 
# git add /function/6267978d69d28205eaa76baa/index* 

- Dependencies 
# git add /function/6267978d69d28205eaa76baa/package* 

- Schema(name,triggers,timeout etc.) 
# git add /function/6267978d69d28205eaa76baa/schema*
